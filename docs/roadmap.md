# Fredsidian Roadmap

## V1
- define architecture
- define folder structure
- define note schema
- establish read-mostly retrieval model
- keep assistant memory separate from broader vault memory

## V1.5
- create templates for core note types
- add assistant draft folder conventions
- document writeback rules

## V2
- scoped write-back into assistant folders
- stronger source citations
- structured retrieval ranking

## V3
- optional local API integration
- semantic indexing / embeddings
- richer provenance and confidence scoring

## Non-Goals for Early Versions
- replacing all assistant memory
- editing the whole vault automatically
- full autonomous publishing to external systems
