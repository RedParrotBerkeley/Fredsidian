# 2026-04-09 Fredsidian v1 scope

## Decision
Fredsidian v1 will use a hybrid model: assistant operational memory plus Obsidian graph memory.

## Context
A pure Obsidian replacement would add unnecessary risk and blur privacy and continuity boundaries.

## Alternatives considered
- replace assistant memory with Obsidian
- delay Obsidian entirely

## Why this was chosen
It preserves what already works while adding linked long-term context.

## Risks
Retrieval and write boundaries must stay explicit.

## Follow-up checks
- define retrieval rules
- define note schema
- decide implementation date
